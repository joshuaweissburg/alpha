The vision for Splotch

Companies run on thousands of processes. Most are not documented. That's fine.

But a few of these flows are critical.

///

Most companies struggle to bring their mission critical flows into focus.

Example candidates

Core user journeys
Ops procedures
System architecture
API integrations
Marketing & sales playbooks

Wait, aren't there already tools for this?

Quite a few! But they all ask the user to translate logic into visual structure to create a flowchart. The effort required to create and maintain flowcharts makes their use relatively rare.

In Splotch, all you need is text

We automate the translation of meaning into visual structure.

The big question is... what kinds of critical flows most need automated mapping?

candidate #1

understand the current state

  • Persona: You're a new hire; you joined a new team; you're starting a new project.
  • Task: You're doing a CUJ or architecture audit. You need to figure out how things work now so you have context for the project.
  • Problem: tldr: you gotta read (a lot) of docs (if they exist at all).

idea: map the docs

Transform existing documentation in Jira/Github/Linear/docs into birds-eye-view flows that update as the docs change. If docs don't exist, create a map by typing up what you learn. Add or edit as your understanding evolves, and your map evolves too.

candidate #2

understand the future state

  • Persona: You're a PM, engineer, designer, ops, sales or marketing manager.
  • Task: Design a new feature, campaign or playbook.
  • Problem: PRDs, marketing plans and ops manuals are hard to parse quickly and skip over edge cases. You try to get people to weigh in, but you spend most of the time bogged down in details.

idea: map the future state

Start with a text descriptions of user stories and key milestones. Add edits by text as you uncover requirements and edge cases. Circulate to your team for confirmation. Now you have a reference that gives context to discussions.

candidate #3

collaborate from a shared map

  • Persona: You're a PM, engineer, designer, ops, sales or marketing manager.
  • Task: You need to get buy-in on a project that is plagued with ambiguities.
  • Problem: You're in meetings all day, but you're not getting closer to a shared picture of how things should work.

idea: compare diffs

Github lets engineers propose, discuss, revert, rework and finalize code by showing the differences between proposed solutions. Splotch could help all teams do the same.
