The big question is...

what kinds of critical flows need to be mapped?

opportunity #1

pain: understand the current state

  • Persona: You're a new hire; you joined a new team; you're starting a new project.
  • Task: You're doing a CUJ, ops or architecture audit. You need to figure out how things work now so you have context for the project.
  • Problem: tldr: you gotta read (a lot) of docs (if they exist at all). Once you do get a handle on how a process works, it stays in your head. The next person has to repeat this.

solution: map the docs

Transform existing documentation in Jira/Github/Linear/docs into birds-eye-view flows that update as the docs change. If docs don't exist, create a map by typing up what you learn. Add or edit as your understanding evolves, and the map evolves too--for everyone.

opportunity #2

pain: identify what's missing in a flow

  • Persona: You're a PM, engineer, designer, ops, sales or marketing manager.
  • Task: You need to create complex flows that account for all the edge cases.
  • Problem: You need to represent numerous conditions, edge cases and permutations but these will take time to identify and integrate into the flow.

solution: Splotch AI fills in the details

While you think about the objectives of the flow, Splotch will help you find edge cases, uncover end states and contradictions in your flow. As we learn more about your stack, we auto-populate using this context.

opportunity #3

pain: co-create the future state

  • Persona: You're a PM, engineer, designer, ops, sales or marketing manager.
  • Task: Design a new feature, campaign or playbook.
  • Problem: PRDs, marketing plans and ops manuals are hard to parse quickly and often miss edge cases. When colleagues weigh in, you spend most of the time bogged down in marginal details because they can't see a full picture of the desired future state.

solution: map the future state

Start with a text description of your flow and key milestones. Add edits by text as you uncover requirements and edge cases. Circulate to your team so they can suggest edits. Now you have a map of the project that you've built together, one decision at a time.

p.s. Engineering has been doing this for a while with code, on Github.
